## Hook Architecture Audit

Current: **39 hooks across 9 events**. PreToolUse [Bash] runs **6 hooks serially** = 25вҖ“100ms overhead per Bash call.

```
PreToolUse [Bash] вҖ” serial execution order:
  1. hyperstack-pretool.sh   (WebFetch intercept, exit 0 for non-web)
  2. rtk-rewrite.sh          (RTK auto-rewrite via Rust binary) вҶҗ CANONICAL
  3. ctx-optimizer.sh        (blocks large-output bash)
  4. context-mode hook       (MCP routing enforcement)
  5. tokenguard-auto.sh      (RTK rewrite v2.0) вҶҗ REDUNDANT with #2
  6. shellfirm               (destructive command guard)

SessionStart: 11 hooks (costly вҖ” 11 process forks at startup)
```

### Optimizations

**A. Remove `tokenguard-auto.sh` from PreToolUse** вҖ” `rtk-rewrite.sh` delegates to the Rust `rtk rewrite` binary (single source of truth in `src/discover/registry.rs`). `tokenguard-auto.sh` is a Bash reimplementation of the same logic. Remove the duplicate.

**B. `rtk read` вҖ” exclude from usage** вҖ” add rule to `ctx-optimizer.sh` blocking `cat`/`head` in Bash and `rtk read` calls; suggest native `Read` tool.

**C. hyperfetch вҖ” add URL size hint** вҖ” only suggest hyperfetch for non-API-endpoint URLs (skip `/json`, `/get`, `/status/*`, `/health`, `/ping`).

**D. SessionStart вҖ” parallel launch** вҖ” 11 serial hooks can run in parallel with `&` + `wait`. Goal: <500ms total startup overhead.
