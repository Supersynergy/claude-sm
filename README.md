## Hook Architecture Audit

Current: **39 hooks across 9 events**. PreToolUse [Bash] runs **6 hooks serially** = 25‚Äď100ms overhead per Bash call.

```
PreToolUse [Bash] ‚ÄĒ serial execution order:
  1. hyperstack-pretool.sh   (WebFetch intercept, exit 0 for non-web)
  2. rtk-rewrite.sh          (RTK auto-rewrite via Rust binary) ‚Üź CANONICAL
  3. ctx-optimizer.sh        (blocks large-output bash)
  4. context-mode hook       (MCP routing enforcement)
  5. tokenguard-auto.sh      (RTK rewrite v2.0) ‚Üź REDUNDANT with #2
  6. shellfirm               (destructive command guard)

SessionStart: 11 hooks (costly ‚ÄĒ 11 process forks at startup)
```

### Optimizations

**A. Remove `tokenguard-auto.sh` from PreToolUse** ‚ÄĒ `rtk-rewrite.sh` delegates to the Rust `rtk rewrite` binary (single source of truth in `src/discover/registry.rs`). `tokenguard-auto.sh` is a Bash reimplementation of the same logic. Remove the duplicate.

**B. `rtk read` ‚ÄĒ exclude from usage** ‚ÄĒ add rule to `ctx-optimizer.sh` blocking `cat`/`head` in Bash and `rtk read` calls; suggest native `Read` tool.

**C. hyperfetch ‚ÄĒ add URL size hint** ‚ÄĒ only suggest hyperfetch for non-API-endpoint URLs (skip `/json`, `/get`, `/status/*`, `/health`, `/ping`).

**D. SessionStart ‚ÄĒ parallel launch** ‚ÄĒ 11 serial hooks can run in parallel with `&` + `wait`. Goal: <500ms total startup overhead.

## Subagent Token Patterns

Per subagent spawn: **30,000 tokens** ($0.45 Opus). Scale fast.

| N agents | Raw cost | Caveman-optimized | ctx_batch replaces all |
|----------|---------:|------------------:|----------------------:|
| 1 | 30,000t | 25,450t | **500t** |
| 3 | 90,000t | 76,350t | **500t** |
| 5 | 150,000t | 127,250t | **500t** |
| 10 | 300,000t | 254,500t | **500t** |
| 20 | 600,000t | 509,000t | **500t** |

`ctx_batch_execute` replaces research subagents entirely ‚ÄĒ **280x cheaper** than spawning an Explore agent.

### When to spawn vs when to batch

| Task | Spawn subagent? | Optimization |
|------|----------------|-------------|
| Research / grep / fetch | **NEVER** | `ctx_batch_execute` ‚ÄĒ 280x cheaper |
| Parallel code execution | YES, minimal ctx | Pass task spec only (caveman prompt) |
| Web scraping N URLs | YES + hyperfetch | Each agent: `hyperfetch --extract "..."` |
| Long builds | YES, `background=True` | `rtk cargo build` inside agent |
| Code review | YES, structured output | Demand JSON result back, not prose |
| Security scan | YES, isolated worktree | Pass file list, not full parent context |

### Subagent prompt optimization rules

```
1. ctx_batch_execute first ‚ÄĒ eliminates 80% of research subagent needs
2. Pass caveman-mode prompts ‚Üí 35% smaller task spec to each agent
3. Demand structured output (JSON/bullets) ‚ÄĒ not prose
4. Never pass full parent context ‚ÄĒ slice only what agent needs
5. use background=True for independent parallel work
6. hyperfetch --extract for web tasks ‚Üí skip spawning researcher agent
7. RTK wraps bash inside subagents too (they run Bash independently)
```
